# FaceClock

Учёт прихода и ухода с работы по распознаванию лица, с многоуровневой защитой
от недобросовестных отметок. Две роли: **сотрудник** и **администратор**.

- **Дизайн:** вариант A «Ясный день» (светлый корпоративный).
- **Мобильный клиент:** Flutter (`app/`).
- **Backend:** Node.js + встроенный SQLite (`server/`).

## Состав репозитория
```
docs/ТЗ.md                     техническое задание (раздел 6 — защита от махинаций)
designs/варианты-дизайна.html  3 варианта UI (выбран A)
server/                        REST API + вся логика защиты (Node.js)
app/                           мобильное приложение (Flutter)
```

## Backend — запуск и проверка
```bash
npm install
npm run seed       # создаёт админа, сотрудника и рабочие места
npm start          # http://localhost:3000
npm test           # логика API: 32 проверки (роли, QR, блокировка, табель, напоминания)
npm run test:face  # распознавание/дубли/живость/офлайн на реальных фото: 14 проверок
```
Тестовые входы: `admin / admin123` (админ), `aziz / aziz123` (сотрудник).

**Распознавание лица считает сервер** (`server/faceEmbed.js`, TensorFlow +
face-api): backend сам вычисляет эмбеддинг из загруженного фото и сравнивает с
эталоном — вектору от клиента не доверяем. Модели берутся из npm-пакета,
вендорить в git ничего не нужно. Отключить/включить: `FACECLOCK_SERVER_EMBED=off`.

Покрыто тестами: серверное распознавание и различение людей, liveness с
одноразовым nonce и анти-replay, геозона, привязка устройства, риск-скоринг,
очередь аномалий, ролевой доступ.

## Мобильное приложение (Flutter)
Сборка выполняется на вашей машине с Flutter SDK — см. **`app/README.md`**.
Кратко: `cd app && flutter create . && flutter pub get && flutter run`,
затем настроить права (камера/гео), положить модель `mobilefacenet.tflite`
и указать адрес сервера на экране входа.

## Как устроена защита от «отметок за друга» (ТЗ, раздел 6)
| Уровень | Механизм |
|---------|----------|
| Живой человек | **Живость проверяет сервер**: анализирует присланные кадры (моргание — по EAR, поворот — по ландмаркам, улыбка — по эмоции), сверяет со случайно выданным действием; одноразовый nonce, анти-replay |
| Именно ты | Эмбеддинг лица считает **сервер** из фото (не доверяем клиенту); **детекция дублей** — одно лицо нельзя привязать к двум сотрудникам; привязка устройства |
| На месте | Геозона (радиус рабочего места), **динамический QR на проходной** (код меняется каждые 30 с), серверное время, флаги mock-локации |
| Контроль | Риск-скоринг, очередь аномалий (спорные отметки — на подтверждение руководителю), фото-аудит |
| Доступ | Блокировка перебора пароля (5 попыток → лок), rate-limit, security-заголовки |

Переключатели окружения: `FACECLOCK_SERVER_EMBED=off` (распознавание на клиенте),
`FACECLOCK_SERVER_LIVENESS=off` (живость по сигналу клиента). По умолчанию — «максимум».

## Функции
- **Табель** (№6): часы, опоздания, переработки за период; экспорт в CSV
  (`GET /api/admin/timesheet?format=csv`). Правила смены — на рабочем месте
  (`work_start`, `norm_hours`).
- **Push-напоминания** (№7): «не отметили приход/уход». Логика — на сервере
  (`server/push.js`), планировщик по `FACECLOCK_REMINDERS=on`. Отправка через
  FCM (`FCM_SERVER_KEY`) или лог в деве. Клиент регистрирует токен
  `POST /api/push/token` (нужна настройка Firebase в приложении).
- **Офлайн-отметки** (№8): нет сети — отметка снимается, сохраняется на устройстве
  и досылается на `POST /api/checkin/offline` при связи; такие отметки всегда идут
  в очередь на проверку (время — со слов устройства).

## Роли
- **Сотрудник:** отметка прихода/ухода по лицу, статус смены, история, офлайн-очередь.
- **Администратор:** управление списком сотрудников (создание/редактирование/
  удаление, сброс лица и устройства), рабочие места, геозоны и **QR-терминал**,
  разбор аномалий, **табель с экспортом**, обзорная статистика.
